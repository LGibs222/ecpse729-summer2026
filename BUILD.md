# ECPSE 729 — Course Site Build

This site is generated by a Python script that wraps shared chrome (head, header, nav, footer) around per-page bodies. You edit one place, all pages update.

## Files at a glance

```
729_course_site/
├── build.py                    ← the build script (templates + data live here)
├── BUILD.md                    ← this file
├── styles.css                  ← all styling (single source of truth)
├── favicon.svg                 ← site favicon (the fleur in Knight red)
├── _src/
│   └── pages/
│       ├── index.body.html     ← home page body (between <main> tags)
│       └── module-01.body.html ← Module 01 body
│
├── index.html                  ← GENERATED — do not edit by hand
├── module-01.html              ← GENERATED — do not edit by hand
├── module-02.html              ← GENERATED — auto-stub (no body file yet)
├── module-03.html              ← GENERATED — auto-stub
└── module-04.html              ← GENERATED — auto-stub
```

## How to build

Open a terminal in this folder and run:

```bash
python build.py
```

You'll see:

```
Building site in C:\...\729_course_site
Last updated: 2026 / 05 / 05

  built  index.html            (body file)
  built  module-01.html        (body file)
  built  module-02.html        (STUB)
  built  module-03.html        (STUB)
  built  module-04.html        (STUB)

Done — 5 page(s) generated.
```

`(body file)` means the page has a hand-written body in `_src/pages/`. `(STUB)` means it's auto-generated from MODULES data because no body file exists yet.

## How to add or edit content

### Change shared chrome (header, nav, footer, head meta)
Edit `build.py` → `PAGE_TEMPLATE` (top of file). Re-run `python build.py`.

### Change a module summary on the home page
Edit `build.py` → `MODULES` list. Each module entry has `title_html`, `summary_html`, `meta_pips`, `theme`, etc. Re-run `python build.py`.

### Lock or unlock a module
In `build.py` → `MODULES` list, change the module's `"status"` from `"locked"` to `"open"` (or vice versa). Re-run.

- `open` modules show as a clickable card on the home page and as a real `<a>` link in nav.
- `locked` modules show as muted, non-clickable cards and as a `<span class="nav-locked">` in nav.

### Write Module 02 (or 03, 04) content
1. Create a new file: `_src/pages/module-02.body.html`
2. Paste your module body content (everything that goes inside `<main id="main">`...`</main>`). Use `module-01.body.html` as a reference for structure (hero, learning objectives, resources, assignments, closer).
3. Run `python build.py`. The next build will pick up your body file and replace the auto-generated stub.

### Add a new page (e.g., a Syllabus page)
1. Add an entry to the `EXTRA_NAV` or `PAGES` list in `build.py`.
2. Create the body file at `_src/pages/<slug>.body.html`.
3. Re-run.

### Replace placeholder URLs (Brightspace, Edpuzzle, etc.)
For now, those are inline in the body files. To replace one:

1. Open `_src/pages/module-01.body.html` (or wherever).
2. Find the `<span class="btn ... btn-locked" aria-disabled="true">Open in Edpuzzle (link pending)</span>`.
3. Replace it with a real anchor: `<a href="https://edpuzzle.com/..." class="btn btn-edpuzzle" target="_blank" rel="noopener">Open in Edpuzzle ↗</a>`.
4. Re-run the build.

(Same pattern for any other locked link.)

## How the build works (one paragraph)

`build.py` defines:
- `PAGE_TEMPLATE` — the full HTML wrapper with `{placeholders}` for title, description, OG tags, header text, nav, body, footer text.
- `MODULES` — list of modules with metadata (title, summary, lock state, etc.).
- `PAGES` — list of pages to generate. The home page is hand-defined; module pages are auto-added from MODULES.

For each page, the script reads its body file (or generates a stub from module data if no file exists), substitutes `{{MODULE_CARDS}}` and `{{LAST_UPDATED}}` placeholders inside the body, renders the nav for the current page, and writes the final HTML.

## When to re-run the build

Whenever you change:
- `build.py` (any data or template change)
- Any file in `_src/pages/`

You do NOT need to re-run for:
- `styles.css` changes (referenced by `<link>` — just refresh the browser)
- `favicon.svg` changes (same — just refresh)

## "I don't want to use the build anymore"

Delete `build.py`, `BUILD.md`, and `_src/`. The generated HTML files remain and are plain, self-contained HTML you can edit directly. The build is opt-in.

## Local preview

```bash
python -m http.server 8729
```

Then open <http://localhost:8729/> in a browser. Or use the preview profile already configured in `.claude/launch.json` (the project root).

## Hosting in Brightspace

This site is plain static HTML/CSS — no server-side anything. To embed:
- Zip the entire folder (excluding `_src/`, `build.py`, `BUILD.md`, `.claude/`) and upload as a content package, OR
- Host on any static host (GitHub Pages, Netlify, etc.) and embed the URL via Brightspace Iframe / LTI / external link.
